# food_photo — 食品写真撮影サービス LP

売れる食品写真の撮影スタジオ（フードフォト）サービスのランディングページ。
ビルド不要の静的サイト。`index.html` をブラウザで開けばそのまま表示できる。

このリポジトリは **2人で共同編集**する前提。お互い Claude / Claude Code で編集するので、
下の「共同編集ルール」を必ず守ること（正本を割らないため）。

---

## ファイル構成

| パス | 中身 |
|---|---|
| `index.html` | LP本体（HTML＋CSS一体）。**これが正本** |
| `images/` | 掲載写真（AI生成の食品写真15点。下表） |
| `README.md` | このファイル。方針・差し替えリスト・プロンプト集 |

### `images/` の内訳
- ヒーロー候補：`hero-food-photo.png` / `hero-food-photo-v2.png` / `hero-steak.png` / `hero-steak-glam.png`

- ギャラリー：`gallery-bento`(弁当) / `deli`(惣菜) / `drink`(ドリンク) / `meat`(肉) / `noodles`(麺) / `seafood`(海鮮) / `seasonal`(季節) / `sweets`(スイーツ)`.png`

## プレビュー方法

`index.html` をダブルクリックしてブラウザで開くだけ。スマホ表示を見る場合は
Chrome の開発者ツール（⌘+Option+I → デバイスツールバー）で幅375pxにする。

## LPの構成（この順番＝コンバージョンの型。崩さない）

1. ファーストビュー（実績数字）
2. 課題提起 —「その写真、…」
3. 解決策（Before/After・ワークフロー）—「売れる見せ方には、…」
4. 選ばれる3つの理由
5. 実績（数字＝品質の証明）
6. お客様の声
7. 撮影の流れ（ご依頼から納品までワンストップ）
8. 料金（撮影は1カットから）
9. 競合比較表（費用が分かりやすい）
10. 撮影実績ギャラリー
11. CTA（お見積り・ご相談は無料）

## デザイン方針（触るときの共通認識）

- **世界観**：食品カタログらしい温かみのある**紙の質感**。テンプレっぽくしない
- **配色**
  - シズル（アクセント）：`#E64E1B` / `#E8804B` / `#F6A06E`（濃→淡のオレンジ）
  - 紙の下地：`#FAF5EE` / `#F3E9DB` / `#F0D9B8`
- **フォント**（Google Fonts読み込み済み）：見出し `Anton` ／本文 `Noto Sans JP` ／飾り `Noto Serif JP`
- **訴求の核**：「長年の撮影力 × 食品特化ノウハウ × ワンストップ（撮影・調理・スタジオ・備品込み）」
- **効かせどころ**：数字を大きく／Before-Afterで一目で価値を伝える／問い合わせCTAは複数配置（意図的。減らさない）

## 共同編集ルール（2人・両方Claude 前提）★重要

1. **正本は1つ = このリポジトリの `index.html`**。ローカルにコピーを増やさない。
2. **編集の前に必ず `git pull`**（Claude Codeなら「最新を取り込んで」）。
   古い状態から編集すると相手の変更を巻き戻す。
3. **Claudeには常に最新の `index.html` を編集させる**（Claude Codeはリポジトリを直接見るのでOK）。
4. **担当を分けるか、着手を宣言**：セクションで分担するか「今から料金セクション触ります」と一声。
   同じ箇所の同時編集は避ける。
5. **こまめにコミット＆プッシュ**。1行でいいので「何を変えたか」を残す（例：`料金表を3プランに変更`）。
6. **プッシュ前にブラウザで表示＆スマホ幅を確認**してから共有。

### Claude Code での基本の流れ（コマンドは覚えなくてOK、日本語で頼めばやってくれる）
```
1.「最新を取り込んで」                       → git pull
2. 編集を依頼
3.「〇〇を変えたのでコミットしてプッシュして」  → git add / commit / push
```
